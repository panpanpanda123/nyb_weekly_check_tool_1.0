# 测试"当天反复"功能流程

## 问题原因
找到了！快照的 `snapshot_date` 字段是 `DateTime` 类型，但代码中用 `date.today()` (date类型) 去比较，导致：
1. 下午导入时，重复检查失败（类型不匹配）
2. 每次都创建新的PM快照，但查询时又找不到
3. "当天反复"判定永远无法触发

### 2. 清空今天的数据（重新开始测试）
```bash
# 删除今天的快照和处理记录
python3 -c "
from shared.database_models import *
from datetime import datetime, date
import os

DATABASE_URL = os.getenv('DATABASE_URL', 'postgresql://postgres:postgres@127.0.0.1:5432/configurable_ops')
engine = create_db_engine(DATABASE_URL, echo=False)
SessionFactory = create_session_factory(engine)
session = SessionFactory()

today_start = datetime.combine(date.today(), datetime.min.time())
today_end = datetime.combine(date.today(), datetime.max.time())

# 删除今天的快照
deleted_snapshots = session.query(EquipmentStatusSnapshot)\
    .filter(EquipmentStatusSnapshot.snapshot_date >= today_start)\
    .filter(EquipmentStatusSnapshot.snapshot_date <= today_end)\
    .delete(synchronize_session=False)

# 删除今天的处理记录
deleted_processing = session.query(EquipmentProcessing)\
    .filter(EquipmentProcessing.processed_at >= today_start)\
    .delete(synchronize_session=False)

session.commit()
print(f'删除 {deleted_snapshots} 条快照')
print(f'删除 {deleted_processing} 条处理记录')
session.close()
"
```

## 为什么之前测试失败？

1. **PM快照从未真正创建**
   - 代码以为已经存在，跳过创建
   - 实际上类型不匹配，查询失败

2. **恢复快照脚本无效**
   - 即使手动创建了快照
   - 查询逻辑仍然有bug，找不到PM快照

3. **Gunicorn缓存问题**
   - 4个worker进程
   - 需要完全重启才能加载新代码
   - `sudo systemctl restart review-viewer`
