# 活动参与度重复数据修复说明

## 问题描述

在活动参与度监控系统中，发现同一个门店会显示多条数据，导致数据重复。

## 问题原因

1. 数据库表结构问题：`promo_participation` 表使用自增 `id` 作为主键，而不是 `store_id`
2. 导入逻辑问题：之前只清空"同一日期"的数据，如果新文件包含多天数据但日期不同，会导致重复

## 解决方案

### 1. 修改数据库表结构

将 `store_id` 改为主键，确保每个门店只能有一条记录：

```python
# 修改前
id = Column(Integer, primary_key=True, autoincrement=True)
store_id = Column(String(50), nullable=False)

# 修改后
store_id = Column(String(50), primary_key=True)
```

### 2. 修改导入逻辑

每次导入时清空所有旧数据，以最新上传的文件为准：

```python
# 修改前：只清空同一日期的数据
deleted = session.query(PromoParticipation)\
    .filter(PromoParticipation.data_date == data_date)\
    .delete(synchronize_session=False)

# 修改后：清空所有旧数据
deleted = session.query(PromoParticipation).delete(synchronize_session=False)
```

1. 迁移脚本会自动保留每个门店最新的一条记录
