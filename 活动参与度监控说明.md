# 活动参与度监控系统使用说明

## 📋 功能概述

活动参与度监控系统用于展示和监控门店的活动参与情况，帮助区域经理督促门店提升活动参与度。

## 🎯 核心功能

### 1. 数据展示
- 门店ID和名称
- 战区和区域经理
- 订单量（F列）
- 权益卡销量（I列）
- 活动套餐销量（J列）
- 活动参与度（K列）

### 2. 筛选功能
- 按战区筛选
- 按区域经理筛选（级联）
- 门店搜索（ID或名称）
- 多种排序方式：
  - 活动参与度（默认）
  - 订单量
  - 权益卡销量
  - 活动套餐销量
- 升序/降序切换

### 3. 数据导出
- 导出Excel文件
- 包含所有筛选后的数据
- 适合进一步分析

---

## 📂 数据文件准备

### 文件1：活动参与度Excel
**位置**：`promo_data/活动参与度MM月DD日.xlsx`

**示例**：`活动参与度3月9日.xlsx`

**说明**：
- 每天更新，日期会变
- 脚本会自动选择最新的文件
- 必须包含以下列：
  - A列：门店ID
  - F列：订单量
  - I列：权益卡销量
  - J列：活动套餐销量
  - K列：活动参与度

### 文件2：活动门店表
**位置**：`promo_data/3月活动门店表.xlsx`

**说明**：
- 参加3月活动的门店列表
- 第一列为门店ID
- 用于筛选参加活动的门店

## 💡 使用技巧

### 技巧1：快速找到低参与度门店
1. 选择自己负责的战区
2. 选择自己的名字
3. 排序方式选择"活动参与度"
4. 排序顺序选择"升序"
5. 最前面的就是参与度最低的门店

### 技巧2：关注重点指标
- **订单量低**：门店整体营业情况不佳
- **权益卡销量低**：会员营销不到位
- **活动套餐销量低**：活动推广不力
- **活动参与度低**：综合表现不佳，需要重点关注

### 技巧3：导出数据分析
1. 设置好筛选条件
2. 点击"导出"按钮
3. 下载Excel文件
4. 可以进一步分析趋势、对比等

## 🔧 数据关联逻辑

### 关联流程
```
活动参与度Excel (A列门店ID)
    ↓
筛选：3月活动门店表 (第一列门店ID)
    ↓
关联：数据库whitelist表 (匹配区域经理)
    ↓
展示：网页列表
```

### 数据匹配规则
1. 从"活动参与度Excel"读取所有门店数据
2. 用"3月活动门店表"筛选出参加活动的门店
3. 用门店ID关联数据库whitelist表，获取：
   - 门店名称
   - 战区
   - 区域经理
4. 组合数据后存入数据库
5. 网页从数据库读取并展示

## ⚠️ 注意事项

### 1. 文件命名
- 活动参与度文件必须包含"活动参与度"关键词
- 文件名中必须包含日期（如：3月9日）
- 支持格式：`活动参与度3月9日.xlsx` 或 `活动参与度2026年3月9日.xlsx`

### 2. 数据更新
- 每次导入会清空同一日期的旧数据
- 建议每天只导入一次
- 如果需要重新导入，直接运行脚本即可

### 3. 活动门店表
- 如果没有活动门店表，会导入所有门店
- 建议提供活动门店表，只展示参加活动的门店

### 4. 区域经理匹配
- 必须先导入whitelist（门店白名单）
- 如果门店在whitelist中不存在，区域经理字段会为空
- 建议定期更新whitelist

## 🐛 常见问题

### Q1：导入后没有数据？
**A**：检查以下几点：
1. Excel文件是否在 `promo_data` 文件夹中
2. 文件名是否包含"活动参与度"
3. 活动门店表是否存在且格式正确
4. 查看导入日志：`logs/promo_import.log`

### Q2：区域经理显示为空？
**A**：门店在whitelist中不存在，需要：
1. 更新whitelist数据
2. 重新导入活动参与度数据

### Q3：数据日期不对？
**A**：检查文件名中的日期格式，支持：
- `活动参与度3月9日.xlsx`
- `活动参与度2026年3月9日.xlsx`

### Q4：排序不准确？
**A**：活动参与度是百分比字符串（如"85%"），系统会自动转换为数字排序
