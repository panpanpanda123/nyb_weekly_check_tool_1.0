# 设备异常监控系统 - 简化使用指南

## 🎯 快速开始（3步）

### 管理员：每天导入数据

```bash
# 1. 上传Excel文件到服务器
scp equipment_status/*.xlsx root@139.224.200.133:/var/www/nyb_weekly_check_tool_1.0/equipment_status/

# 2. 连接服务器
ssh root@139.224.200.133

# 3. 运行导入
cd /var/www/nyb_weekly_check_tool_1.0
python3 import_equipment_data.py
```

### 区域经理：手机查看处理

1. 打开：https://weeklycheck.blitzepanda.top/equipment
2. 选择自己的名字
3. 点击"已恢复"或"未恢复"

---

## 📋 核心功能

### 1. 自动判定"多次出问题"

| 情况 | 判定结果 |
|------|---------|
| 上午标记"已恢复" + 下午还是离线 | 🔴 多次出问题（当天反复） |
| 上午+下午都离线 + 没有处理 | 🔴 多次出问题（未及时处理） |
| 5天内≥3次异常 | 🔴 多次出问题（5天3次） |
| 10天内≥4次异常 | 🔴 多次出问题（10天4次） |

**详细说明**：查看 `设备异常判定逻辑说明.md`

### 2. 预计恢复日期

- 点击"未恢复"，填写预计恢复日期
- 在期限内不会重复提示
- 减少区域经理重复操作

### 3. 自动清理

- 只保留最近10天数据
- 自动删除过期记录
- 节省存储空间

## 🔧 优化建议

### 建议1：简化导入流程 ⭐⭐⭐

**当前**：手动上传文件 + SSH连接 + 运行脚本

**优化方案**：
1. **Web上传界面**：在网页上直接上传Excel文件
2. **定时任务**：设置cron自动从共享文件夹读取
3. **API接口**：提供API，其他系统直接推送数据

**推荐**：Web上传界面（最简单）

### 建议2：增加通知功能 ⭐⭐

**当前**：区域经理需要主动查看

**优化方案**：
1. **企业微信通知**：每天自动推送异常门店
2. **邮件通知**：发送每日报告
3. **短信通知**：紧急情况（当天反复）

**推荐**：企业微信通知（区域经理都在用）

### 建议3：优化移动端体验 ⭐⭐⭐

**当前**：已经适配手机，但可以更好

**优化方案**：
1. **一键处理**：长按门店卡片快速标记
2. **语音输入**：未恢复原因支持语音
3. **拍照上传**：支持上传现场照片

**推荐**：一键处理（提升效率）

### 建议4：增加数据分析 ⭐

**当前**：只能导出Excel手动分析

**优化方案**：
1. **趋势图表**：异常数量趋势
2. **排行榜**：问题最多的门店/区域经理
3. **对比分析**：战区之间对比

**推荐**：趋势图表（直观）

### 建议5：简化配置管理 ⭐

**当前**：需要编辑Python文件修改规则

**优化方案**：
1. **Web配置界面**：在网页上修改规则
2. **配置文件**：使用JSON/YAML配置
3. **环境变量**：通过环境变量配置

**推荐**：保持现状（改动频率低）

1. **功能完整**：覆盖所有核心需求
2. **逻辑清晰**：判定规则明确

## 💡 使用技巧

### 技巧1：使用筛选器

- 点击统计卡片快速筛选
- 组合使用战区+区域经理
- 搜索框支持门店ID和名称

### 技巧2：预计恢复日期

- 短期问题（1-3天）：填写预计恢复日期
- 长期问题（>7天）：不填日期，持续提示
- 到期后自动恢复提示

### 技巧3：导出数据

- 导出包含所有统计信息
- 可用于绩效考核
- 可用于处罚依据

### 技巧4：查看历史

- 异常时间点显示最近5次
- 未处理日期显示所有未处理的天
- 用于追溯问题
