# 设备异常功能部署指南

## 📋 服务器信息

- **服务器地址**: `root@blitzepanda.top` 或 `root@你的服务器IP`
- **项目位置**: `/var/www/nyb_weekly_check_tool_1.0`
- **服务名称**: `review-viewer`
- **监听端口**: `8000`
- **访问地址**: `https://weeklycheck.blitzepanda.top`

## 🚀 首次部署

### 1. 提交代码到Git

```bash
git add .
git commit -m "添加设备异常监控功能"
git push origin main
```

### 2. 在服务器上部署

SSH登录服务器：
```bash
ssh root@blitzepanda.top
```

执行部署脚本：
```bash
cd /var/www/nyb_weekly_check_tool_1.0
chmod +x deploy_equipment.sh
./deploy_equipment.sh
```

部署脚本会自动：
- ✅ 拉取最新代码
- ✅ 检查设备数据文件
- ✅ 导入设备数据
- ✅ 重启服务
- ✅ 测试功能

### 3. 访问测试

打开浏览器访问：
- 周清审核：`https://weeklycheck.blitzepanda.top/`
- 门店评级：`https://weeklycheck.blitzepanda.top/rating`
- 设备异常：`https://weeklycheck.blitzepanda.top/equipment`

## 🔄 更新设备数据（日常使用）

### 方法一：使用自动脚本（推荐）

1. **下载新表格**，放到 `equipment_status/` 文件夹：
   - `牛约堡集团_点餐设备表_*.xlsx`
   - `202*.xlsx`（机顶盒）
   - `牛约堡集团_门店档案_*.xlsx`

2. **修改脚本配置**（首次使用）：
   
   打开 `更新设备数据.bat`，修改第7-8行：
   ```batch
   set SERVER=root@blitzepanda.top
   set PROJECT_DIR=/var/www/nyb_weekly_check_tool_1.0
   ```

3. **运行脚本**：
   ```bash
   更新设备数据.bat
   ```

4. **完成**！访问网站查看最新数据

### 方法二：手动操作

```bash
# 1. 上传文件
scp equipment_status/*.xlsx root@blitzepanda.top:/var/www/nyb_weekly_check_tool_1.0/equipment_status/

# 2. SSH登录
ssh root@blitzepanda.top

# 3. 导入数据
cd /var/www/nyb_weekly_check_tool_1.0
python3 import_equipment_data.py

# 4. 完成！
```

## 📝 重要说明

- ✅ 设备异常数据会被清空并重新导入

## 🔧 故障排查

### 问题1：页面404

```bash
# 检查服务状态
systemctl status review-viewer

# 重启服务
systemctl restart review-viewer
```

### 问题2：数据为空

```bash
# 检查数据是否导入
ssh root@blitzepanda.top
cd /var/www/nyb_weekly_check_tool_1.0
python3 import_equipment_data.py
```

### 问题3：筛选选项为空

- 确认设备数据已导入
- 确认whitelist已存在
- 查看服务日志：`journalctl -u review-viewer -n 100`

### 问题4：上传文件失败

- 检查服务器地址是否正确
- 确认SSH密钥已配置
- 尝试手动SSH登录测试
