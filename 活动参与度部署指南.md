# 活动参与度功能部署指南

## 📋 部署前准备

### 1. 确认代码已更新
在服务器上执行：
```bash
cd /var/www/nyb_weekly_check_tool_1.0
git pull
```

### 2. 检查文件完整性
确认以下文件已存在：
- `import_promo_data.py` - 数据导入脚本
- `deploy_promo.sh` - 部署脚本
- `viewer/api_promo.py` - API接口
- `viewer/templates/promoratio.html` - 前端页面
- `viewer/static/promoratio.js` - 前端逻辑
- `viewer/static/promoratio.css` - 样式文件

## 📁 Excel文件放置

### 文件位置
将Excel文件放到以下目录：
```bash
/var/www/nyb_weekly_check_tool_1.0/promo_data/
```

### 需要的文件
1. **活动参与度数据文件**（每天更新）
   - 文件名格式：`活动参与度MM月DD日.xlsx`
   - 例如：`活动参与度3月9日.xlsx`
   - 脚本会自动选择最新的文件

2. **活动门店表**（固定文件）
   - 文件名：`3月活动门店表.xlsx`
   - 用于筛选参加活动的门店

### 上传文件命令
```bash

scp "3月活动门店表.xlsx" root@139.224.200.133:/var/www/nyb_weekly_check_tool_1.0/promo_data/

# 方法2：在服务器上直接创建目录并上传
ssh root@139.224.200.133
```

如果需要指定文件导入：
```bash

```

## 🐛 故障排查

## 📊 数据说明

### Excel文件格式要求

**活动参与度文件**（活动参与度MM月DD日.xlsx）：
- A列：门店ID
- F列：订单量
- I列：权益卡销量
- J列：活动套餐销量
- K列：活动参与度

**活动门店表**（3月活动门店表.xlsx）：
- 包含参加活动的门店ID列表
- 用于筛选哪些门店参加了活动

### 数据处理流程
1. 读取"活动参与度MM月DD日.xlsx"
2. 与"3月活动门店表.xlsx"关联，筛选参加活动的门店
3. 与数据库whitelist表关联，获取区域经理信息
4. 存入PromoParticipation表
5. 前端展示

## 📝 注意事项

1. **文件命名**：活动参与度文件必须包含日期（如"3月9日"），脚本会自动提取日期
2. **文件覆盖**：可以直接覆盖旧文件，脚本会自动选择最新的
3. **数据更新**：每次导入会清空旧数据，重新导入新数据
4. **热更新**：更新代码后只需重启服务，不影响数据收集
5. **移动优化**：页面已针对手机浏览器优化，区域经理可以用手机查看
