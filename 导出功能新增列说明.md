# 导出功能新增列说明（快速参考）

## 新增5列

| 列名 | 说明 | 用途 |
|------|------|------|
| **近10天处理次数** | 区域经理对该门店该设备的总处理次数 | 查看处理频率，识别反复出问题的门店 |
| **近10天已恢复次数** | 标记"已恢复"的次数 | 评估问题解决效果 |
| **近10天未恢复次数** | 标记"未恢复"的次数 | 识别难以解决的问题 |
| **最近处理记录** | 最近5条处理记录详情 | 查看处理历史和时间分布 |
| **预计恢复期内跳过** | 是否在预计恢复期内被跳过 | 验证预计恢复机制是否正常工作 |

## 快速示例

### 反复出问题的门店
```
近10天处理次数: 8
近10天已恢复次数: 5
近10天未恢复次数: 3
最近处理记录: 03-11 10:30 已恢复；03-10 17:00 未恢复(设备故障)；03-09 11:00 已恢复
预计恢复期内跳过: 否
```
→ 需要重点关注，可能需要更换设备

### 预计恢复期内的门店
```
近10天处理次数: 2
近10天已恢复次数: 0
近10天未恢复次数: 2
最近处理记录: 03-11 10:00 未恢复(等待配件)；03-09 09:30 未恢复(设备故障)
预计恢复期内跳过: 是（预计03-15恢复）
```
→ 等待配件，预计3月15日恢复，期间不重复提示

### 从未处理的门店
```
近10天处理次数: 0
近10天已恢复次数: 0
近10天未恢复次数: 0
最近处理记录: （空）
预计恢复期内跳过: 否
```
→ 可能被忽略，需要提醒区域经理

## 导出方式

### 网页导出
https://weeklycheck.blitzepanda.top/equipment → 点击"导出Excel"

### API导出（可选）
```bash
# 默认统计近10天
curl -o 设备异常.xlsx "https://weeklycheck.blitzepanda.top/api/equipment/export"

# 统计近7天
curl -o 设备异常.xlsx "https://weeklycheck.blitzepanda.top/api/equipment/export?history_days=7"
```

## 验证清单

- [ ] 导出Excel文件
- [ ] 检查新增的5列是否存在
- [ ] 检查"近10天处理次数"数据是否正确
- [ ] 检查"最近处理记录"格式是否正确
- [ ] 检查"预计恢复期内跳过"状态是否准确
- [ ] 验证：已恢复次数 + 未恢复次数 = 处理次数

## 注意事项

1. 默认统计近10天，可通过URL参数调整（7-14天）
2. 处理记录保留10天，超过的会被自动清理
3. "最近处理记录"最多显示5条
4. 预计恢复期内的门店在网页上不显示，但会在导出中标记
