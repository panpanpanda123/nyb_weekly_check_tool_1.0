# 自动化上传方案（可选）

## ⚠️ 重要提示

**推荐使用FinalShell手动上传**，参考《FinalShell使用指南.md》

如果你确实需要自动化，可以参考以下方案。但请注意：
- 需要在脚本中保存密码（安全风险）
- 需要安装额外工具
- 配置相对复杂

---

## 方案1：使用plink + pscp（推荐）

### 步骤1：下载工具

下载PuTTY工具包（包含plink和pscp）：
https://www.chiark.greenend.org.uk/~sgtatham/putty/latest.html

下载 `putty.zip`，解压后将以下文件放到项目根目录：
- `plink.exe`
- `pscp.exe`

### 步骤2：创建密码文件

创建文件 `server_password.txt`，内容为你的服务器密码：
```
你的密码
```

**重要：** 将此文件添加到 `.gitignore`，避免上传到Git！

### 步骤3：修改bat脚本

创建 `自动上传设备数据.bat`：

```batch
@echo off
chcp 65001 >nul
echo ============================================================
echo 自动上传设备数据到服务器
echo ============================================================
echo.

REM 读取密码
set /p PASSWORD=<server_password.txt

REM 服务器信息
set SERVER=139.224.200.133
set USER=root
set PROJECT_DIR=/var/www/nyb_weekly_check_tool_1.0

echo 📁 检查本地数据文件...
if not exist "equipment_status\*在营门店*.xlsx" (
    echo ❌ 未找到在营门店文件
    pause
    exit /b 1
)

echo ✅ 本地文件检查完成
echo.

echo 📤 上传文件到服务器...
echo.

REM 上传在营门店文件
for %%f in (equipment_status\*在营门店*.xlsx) do (
    echo 上传: %%f
    pscp -pw %PASSWORD% "%%f" %USER%@%SERVER%:%PROJECT_DIR%/equipment_status/
)

REM 上传收银设备文件（如果存在）
if exist "equipment_status\牛约堡集团_点餐设备表*.xlsx" (
    for %%f in (equipment_status\牛约堡集团_点餐设备表*.xlsx) do (
        echo 上传: %%f
        pscp -pw %PASSWORD% "%%f" %USER%@%SERVER%:%PROJECT_DIR%/equipment_status/
    )
)

REM 上传机顶盒文件（如果存在）
if exist "equipment_status\202*.xlsx" (
    for %%f in (equipment_status\202*.xlsx) do (
        echo 上传: %%f
        pscp -pw %PASSWORD% "%%f" %USER%@%SERVER%:%PROJECT_DIR%/equipment_status/
    )
)

echo.
echo ✅ 文件上传完成
echo.

echo 🔄 在服务器上导入数据...
plink -pw %PASSWORD% %USER%@%SERVER% "cd %PROJECT_DIR% && python3 import_equipment_data.py"

if errorlevel 1 (
    echo.
    echo ❌ 数据导入失败
    pause
    exit /b 1
)

echo.
echo ============================================================
echo ✅ 数据更新完成！
echo ============================================================
echo.
pause
```

### 步骤4：使用

1. 确保 `server_password.txt` 包含正确的密码
2. 将数据文件放入 `equipment_status/` 目录
3. 双击运行 `自动上传设备数据.bat`

## 方案2：使用SSH密钥（更安全）

### 步骤1：生成SSH密钥

打开PowerShell，执行：
```powershell
ssh-keygen -t rsa -b 4096
```

一路回车，不设置密码。

### 步骤2：上传公钥到服务器

```powershell
type $env:USERPROFILE\.ssh\id_rsa.pub | ssh root@139.224.200.133 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

输入一次服务器密码。

### 步骤3：测试免密登录

```powershell
ssh root@139.224.200.133
```

如果不需要输入密码就能登录，说明配置成功。

## 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| FinalShell手动 | 简单、安全、直观 | 需要手动操作 | ⭐⭐⭐⭐⭐ |
| plink+密码 | 全自动 | 密码明文保存 | ⭐⭐ |
| SSH密钥 | 全自动、安全 | 配置复杂 | ⭐⭐⭐⭐ |

---

## 安全建议

### 如果使用密码方案：
1. ✅ 将 `server_password.txt` 添加到 `.gitignore`
2. ✅ 不要分享包含密码的脚本
3. ✅ 定期更换服务器密码

### 如果使用SSH密钥方案：
1. ✅ 保护好私钥文件 `~/.ssh/id_rsa`
2. ✅ 不要将私钥上传到任何地方
3. ✅ 可以为私钥设置密码保护

---

## .gitignore 配置

确保以下文件不会被提交到Git：

```gitignore
# 密码文件
server_password.txt

# SSH密钥
*.pem
*.key
id_rsa
id_rsa.pub

# PuTTY工具
plink.exe
pscp.exe
putty.exe
```

## 总结

**最推荐的方式仍然是使用FinalShell手动上传**：
- 简单直接
- 安全可靠
- 可以实时看到进度和结果
- 不需要额外配置

如果你确实需要自动化，建议使用SSH密钥方案，避免在脚本中保存密码。

但对于日常使用，手动上传完全够用，而且更灵活。
