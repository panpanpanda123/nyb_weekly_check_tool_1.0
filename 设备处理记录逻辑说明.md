# 设备处理记录逻辑说明

## 问题描述

之前的逻辑错误：下午导入新数据后，上午处理过的门店仍然显示为"已处理"状态（59家），导致无法正确识别新一轮需要处理的门店。

## 修复后的逻辑

### 核心原则
每次导入新数据 = 新一轮检查，处理状态应该重置，让区域经理重新处理。

### 具体实现

#### 1. 上午导入（AM，14点前）
```
清空今天所有的处理记录
→ 所有门店显示"未处理"
→ 区域经理开始处理
```

#### 2. 下午导入（PM，14点后）
```
清空今天下午的处理记录
保留今天上午的处理记录（用于判定"当天反复"）
→ 所有门店显示"未处理"（下午新一轮）
→ 区域经理重新处理
```

#### 3. API显示逻辑
```
上午时段：只显示今天上午的处理记录
下午时段：只显示今天下午的处理记录
→ 确保"已处理/未处理"状态正确
```

### "当天反复"检测逻辑

虽然下午导入时清空了下午的处理记录，但上午的处理记录被保留，用于检测"当天反复"：

```
条件：
1. 上午有异常快照（AM snapshot）
2. 上午标记"已恢复"（AM processing record）
3. 下午又有异常快照（PM snapshot）

结果：
→ 标记为"多次出问题（当天反复）"
→ 重点关注这类门店
```

## 数据流程示例

### 场景1：正常恢复
```
09:00 上午导入 → 门店A异常
10:00 区域经理标记"已恢复"
15:00 下午导入 → 门店A正常（没有异常记录）
结果：门店A不再显示
```

### 场景2：当天反复
```
09:00 上午导入 → 门店A异常
10:00 区域经理标记"已恢复"
15:00 下午导入 → 门店A又异常
结果：门店A显示"未处理"，标记"多次出问题（当天反复）"
```

### 场景3：未及时处理
```
09:00 上午导入 → 门店A异常
（区域经理没有处理）
15:00 下午导入 → 门店A还是异常
结果：门店A显示"未处理"，标记"多次出问题（未及时处理）"
```

### 场景4：预计恢复期内
```
09:00 上午导入 → 门店A异常
10:00 区域经理标记"未恢复"，填写预计恢复日期：3月12日
15:00 下午导入 → 门店A还是异常
结果：门店A不显示（在预计恢复期内被过滤）
```

## 代码位置

### 1. 导入脚本（清空处理记录）
文件：`import_equipment_data.py`
位置：第8节 "创建快照"

```python
if snapshot_period == 'AM':
    # 上午导入：清空今天所有处理记录
    deleted_current = session.query(EquipmentProcessing)\
        .filter(EquipmentProcessing.processed_at >= today_start)\
        .delete(synchronize_session=False)
else:
    # 下午导入：清空今天下午的处理记录，保留上午的
    pm_start = datetime.combine(date.today(), datetime.min.time().replace(hour=AM_PM_BOUNDARY_HOUR))
    deleted_pm = session.query(EquipmentProcessing)\
        .filter(EquipmentProcessing.processed_at >= pm_start)\
        .delete(synchronize_session=False)
```

```python
if current_period == 'AM':
```

### 3. 经常出问题检测（使用保留的上午记录）
文件：`equipment_utils.py`
位置：`is_chronic_store()` 函数

```python
# 检查今天是否有处理记录（包括上午的）
today_processing = session.query(EquipmentProcessing)\
    .filter(EquipmentProcessing.processed_at >= today_start)\
    .first()

# 情况1：上午有异常 + 标记"已恢复" + 下午又有异常 = 当天反复
if am_abnormal and pm_abnormal and today_processing and today_processing.action == '已恢复':
    return True, "多次出问题（当天反复）", {'today_repeat': 1}
```

## 配置说明

相关配置在 `equipment_config.py`：

- `AM_PM_BOUNDARY_HOUR = 14`：上午/下午分界点（14点）
- `PROCESSING_RETENTION_DAYS = 10`：处理记录保留天数
- `SNAPSHOT_RETENTION_DAYS = 10`：快照保留天数
