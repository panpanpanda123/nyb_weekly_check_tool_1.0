# 设备异常功能修改说明

## 📋 修改内容

### 1. 处理选项更新

**旧版本:**
- ✓ 已处理
- ✗ 不配合
- ⚠ 特殊情况(需填写理由)

**新版本:**
- ✓ 已恢复
- ⚠ 未恢复(需填写原因)

### 2. 数据导入行为变更

**旧版本:**
- 每次导入只清空设备异常数据
- 处理记录永久保存

**新版本:**
- 每次导入同时清空设备异常数据和处理记录
- 重新统计,不保留历史处理记录

## 🔧 故障排查

### 问题1:主页设备异常按钮无反应

**原因**:浏览器缓存

**解决**:
- 强制刷新: Ctrl+F5 (Windows) 或 Cmd+Shift+R (Mac)
- 清除浏览器缓存
- 检查浏览器控制台是否有JavaScript错误

### 问题2:处理记录没有被清空

**原因**:导入脚本未更新

**解决**:
```bash
# 检查import_equipment_data.py是否包含清空代码
grep "EquipmentProcessing" import_equipment_data.py

# 应该看到:
# from shared.database_models import EquipmentProcessing
# session.query(EquipmentProcessing).delete()
```

### 问题3:按钮文本没有更新

**原因**:浏览器缓存或服务未重启

**解决**:
```bash
# 重启服务
systemctl restart review-viewer

# 强制刷新浏览器
Ctrl+F5
```
