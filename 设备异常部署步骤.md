# 设备异常功能部署步骤

## 目标配置
- 周清展示：`weeklycheck.blitzepanda.top` （现有，不变）
- 设备异常：`blitzepanda.top/equipment` （新增）

## 部署步骤

### 1. 提交代码到Git

在本地执行：
```bash
git add .
git commit -m "添加设备异常监控功能"
git push origin main
```

### 2. 在服务器上拉取代码

SSH登录服务器：
```bash
ssh root@your-server
```

进入项目目录并拉取代码：
```bash
cd /var/www/nyb_weekly_check_tool_1.0
git pull origin main
```

### 3. 上传设备数据文件

从本地上传Excel文件到服务器：
```bash
# 在本地执行
scp equipment_status/牛约堡集团_点餐设备表*.xlsx root@your-server:/var/www/nyb_weekly_check_tool_1.0/equipment_status/
scp equipment_status/202*.xlsx root@your-server:/var/www/nyb_weekly_check_tool_1.0/equipment_status/
scp equipment_status/牛约堡集团_门店档案*.xlsx root@your-server:/var/www/nyb_weekly_check_tool_1.0/equipment_status/
```

或者在服务器上创建文件夹：
```bash
# 在服务器上执行
mkdir -p /var/www/nyb_weekly_check_tool_1.0/equipment_status
```

### 4. 导入设备数据

在服务器上执行：
```bash
cd /var/www/nyb_weekly_check_tool_1.0
python3 import_equipment_data.py
```

### 5. 重启服务

```bash
systemctl restart review-viewer
systemctl status review-viewer
```

### 6. 配置Nginx（如果需要）

如果你想让设备异常功能在主域名 `blitzepanda.top/equipment` 下访问，需要配置Nginx。

但是，**当前的 `review-viewer` 服务已经运行在 `weeklycheck.blitzepanda.top`**，它包含了所有功能（周清、评级、设备异常）。

#### 方案A：保持现状（推荐）
设备异常功能已经在 `weeklycheck.blitzepanda.top/equipment` 可以访问了，不需要额外配置。

访问地址：
- 周清审核：`https://weeklycheck.blitzepanda.top/`
- 门店评级：`https://weeklycheck.blitzepanda.top/rating`
- 设备异常：`https://weeklycheck.blitzepanda.top/equipment`

#### 方案B：配置主域名反向代理
如果你一定要在 `blitzepanda.top/equipment` 访问，需要修改主域名的Nginx配置。

编辑主域名配置文件：
```bash
nano /etc/nginx/sites-available/blitzepanda.top
```

添加location配置：
```nginx
server {
    listen 80;
    server_name blitzepanda.top www.blitzepanda.top;
    
    # 其他现有配置...
    
    # 设备异常功能代理到weeklycheck服务
    location /equipment {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 静态文件
    location /static {
        proxy_pass http://127.0.0.1:8000;
    }
    
    # API
    location /api/equipment {
        proxy_pass http://127.0.0.1:8000;
    }
}
```

测试并重启Nginx：
```bash
nginx -t
systemctl reload nginx
```

```bash

journalctl -u review-viewer -n 100 -f

tail -f /var/log/nginx/error.log

```

## 注意事项

1. **数据库**：确保PostgreSQL服务正常运行
2. **whitelist**：设备数据需要匹配whitelist，确保whitelist已导入
3. **权限**：确保项目目录有正确的权限
4. **端口**：review-viewer服务运行在8000端口
5. **SSL证书**：如果使用HTTPS，确保证书配置正确

## 故障排查

### 问题1：页面404
- 检查服务是否运行：`systemctl status review-viewer`
- 检查路由是否正确：代码中已添加 `/equipment` 路由

### 问题2：数据为空
- 检查是否导入设备数据：`python3 import_equipment_data.py`
- 检查whitelist是否存在
- 查看数据库：`psql -U postgres -d configurable_ops -c "SELECT COUNT(*) FROM equipment_status;"`

### 问题3：筛选选项为空
- 确认设备数据已导入
- 检查数据库连接
- 查看服务日志

### 问题4：处理功能不工作
- 检查API是否正常：`curl http://localhost:8000/api/equipment/filters`
- 查看浏览器控制台错误
- 检查服务日志
