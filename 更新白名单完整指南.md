# 更新白名单完整指南

## 概述

系统中有两个"白名单"概念，作用不同：

### 白名单1：人员匹配白名单（whitelist.xlsx）
- **文件**：`whitelist.xlsx`
- **作用**：匹配门店的战区、区域经理、运营人员
- **存储**：数据库表 `store_whitelist`
- **影响范围**：
  - ✅ 周清审核：运营人员分配
  - ✅ 门店评级：区域经理筛选
  - ✅ 设备异常：战区、区域经理筛选
- **更新频率**：人员调整时更新

### 白名单2：营业状态白名单（在营门店.xlsx）
- **文件**：`在营门店.xlsx`（文件名包含"在营门店"）
- **作用**：判断门店是否营业中
- **存储**：不存数据库，导入时临时使用
- **影响范围**：
  - ✅ 设备异常：只展示营业中的门店
- **更新频率**：门店营业状态变化时更新

## 更新 whitelist.xlsx 的完整流程

### 场景：区域经理调整了

#### 本地操作

1. **更新本地 whitelist.xlsx**
   ```
   编辑 whitelist.xlsx
   → 修改区域经理列
   → 保存文件
   ```

2. **本地测试（可选）**
   ```bash
   # 如果有本地数据库，可以先测试
   python3 update_whitelist_only.py
   ```

#### 云服务器操作

1. **连接服务器**
   ```bash
   ssh root@139.224.200.133
   ```

2. **进入项目目录**
   ```bash
   cd /var/www/nyb_weekly_check_tool_1.0
   ```

3. **备份旧的 whitelist（可选但推荐）**
   ```bash
   cp whitelist.xlsx whitelist_backup_$(date +%Y%m%d_%H%M%S).xlsx
   ```

4. **上传新的 whitelist.xlsx**
   - 使用 WinSCP 上传到 `/var/www/nyb_weekly_check_tool_1.0/`
   - 覆盖原有的 `whitelist.xlsx`

5. **运行更新脚本**

## 更新 在营门店.xlsx 的流程

### 场景：门店营业状态变化

3. **重新导入设备数据**

   ```bash
   python3 import_equipment_data.py
   ```

4. **验证**
   - 访问设备异常页面
   - 确认只显示营业中门店的设备异常

### 场景：人员调整 + 门店状态变化

1. **准备文件**
   - 更新 `whitelist.xlsx`（人员信息）

   ```bash
   在营门店.xlsx → /var/www/nyb_weekly_check_tool_1.0/equipment_status/
   ```

3. **依次执行**
   ```bash
   ssh root@139.224.200.133

   # 1. 更新人员白名单
   python3 update_whitelist_only.py
   ```

   - 设备异常：战区、区域经理是否正确，门店是否都是营业中

## 常见问题

### Q1: 更新 whitelist 后，设备异常页面的区域经理没变？
**A**: 需要重新导入设备数据：
```bash
python3 import_equipment_data.py
```

### Q2: 更新 whitelist 会影响已处理的设备吗？
**A**: 不会。已处理状态存储在 `equipment_processing` 表中，不受影响。

### Q3: 更新 whitelist 会影响周清审核结果吗？
**A**: 不会。审核结果存储在 `viewer_review_results` 表中，不受影响。

### Q4: 如何确认 whitelist 更新成功？
**A**: 
1. 查看脚本输出的"更新记录数"
2. 访问网页，刷新后查看筛选选项
3. 搜索具体门店，查看人员信息

### Q5: 两个白名单可以不同步更新吗？
**A**: 可以。它们是独立的：
- `whitelist.xlsx` 影响人员匹配
- `在营门店.xlsx` 影响营业状态筛选

### Q6: 如果忘记更新 whitelist，直接导入设备数据会怎样？
**A**: 
- 设备数据会使用旧的人员信息
- 战区、区域经理可能不准确
- 建议先更新 whitelist，再导入设备数据

## 最佳实践

### 1. 定期更新流程
```
每周/每月：
1. 检查人员调整 → 更新 whitelist.xlsx
2. 检查门店状态 → 更新 在营门店.xlsx
3. 先更新 whitelist → python3 update_whitelist_only.py
4. 再导入设备数据 → python3 import_equipment_data.py
```

### 2. 文件管理
```
本地保留：
- whitelist.xlsx（最新版本）
- whitelist_历史版本.xlsx（备份）

云服务器保留：
- whitelist.xlsx（当前使用）
- whitelist_backup_YYYYMMDD_HHMMSS.xlsx（自动备份）
```

### 3. 验证清单
```
更新后验证：
□ 周清审核 - 运营人员正确
□ 门店评级 - 区域经理正确
□ 设备异常 - 战区/区域经理正确
□ 设备异常 - 只显示营业中门店
□ 筛选功能 - 选项列表正确
```

- **whitelist.xlsx**：人员匹配，影响所有功能的人员信息
- **在营门店.xlsx**：营业状态，只影响设备异常的门店筛选
