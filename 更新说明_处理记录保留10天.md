# 更新说明 - 处理记录保留10天

## 更新内容

### 1. 处理记录保留策略调整
- **之前**：处理记录永久保留
- **现在**：处理记录只保留最近10天，自动清理旧数据

### 2. 容错性增强
- ✅ 休息期间缺失日期不会报错
- ✅ 系统自动忽略缺失的日期
- ✅ 只保留最近10个自然日的数据

### 3. 存储优化
- 快照数据：保留10天
- 处理记录：保留10天
- 自动清理：每次导入时清理超过10天的数据

---

## 修改的文件

### 1. `equipment_config.py`
新增配置项：
```python
# 处理记录保留天数
PROCESSING_RETENTION_DAYS = 10
```

### 2. `import_equipment_data.py`
新增清理逻辑：
```python
# 清理旧处理记录
processing_cutoff_date = date.today() - timedelta(days=PROCESSING_RETENTION_DAYS)
deleted_processing = session.query(EquipmentProcessing)\
    .filter(EquipmentProcessing.processed_at < processing_cutoff_date)\
    .delete()
```

### 3. `设备异常监控使用说明.md`
更新文档说明：
- 核心逻辑：数据保留10天
- 导入逻辑：自动清理旧数据

## 注意事项

### 1. 数据迁移
- 首次部署后，会清理所有超过10天的旧数据
- 这是正常的，不会影响功能

### 2. 休息期间
- 如果休息几天没有维护，不会报错
- 系统会自动忽略缺失的日期
- 只保留最近10个自然日的数据

### 3. 配置调整
如需修改保留天数：
```python
# equipment_config.py
SNAPSHOT_RETENTION_DAYS = 7      # 改为7天
PROCESSING_RETENTION_DAYS = 7    # 改为7天
```
