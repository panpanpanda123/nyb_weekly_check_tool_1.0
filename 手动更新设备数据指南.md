# 手动更新设备数据指南

## 概述
本指南说明如何手动更新设备异常数据，支持灵活的导入模式。

## 前提条件
1. 已准备好最新的设备数据文件
2. 已连接到服务器或本地环境

## 数据文件说明

### 必需文件
将以下文件放入 `equipment_status/` 目录：

1. **在营门店文件**（必需）
   - 文件名格式：`*在营门店*.xlsx`
   - 示例：`0305在营门店.xlsx`
   - 用途：判断门店营业状态

2. **收银设备文件**（导入POS时必需）
   - 文件名格式：`牛约堡集团_点餐设备表_*.xlsx`
   - 示例：`牛约堡集团_点餐设备表_20260305_1124_douhuanwei_1772681050478.xlsx`
   - 用途：POS收银系统状态

3. **机顶盒文件**（导入机顶盒时必需）
   - 文件名格式：`202*.xlsx`
   - 示例：`20260305032606-1772681166.xlsx`
   - 用途：机顶盒设备状态

## 导入模式

### 模式1：导入所有设备（默认）
导入POS和机顶盒的所有数据，清空并替换旧数据。

**Windows:**
```cmd
python import_equipment_data.py
```

**Linux/Mac:**
```bash
python3 import_equipment_data.py
```

### 模式2：只导入POS
只导入收银系统数据，不影响机顶盒数据。

**Windows:**
```cmd
python import_equipment_data.py --only-pos
```

**Linux/Mac:**
```bash
python3 import_equipment_data.py --only-pos
```

**使用场景：** 每日检查收银系统状态

### 模式3：只导入机顶盒
只导入机顶盒数据，不影响POS数据。

**Windows:**
```cmd
python import_equipment_data.py --only-stb
```

**Linux/Mac:**
```bash
python3 import_equipment_data.py --only-stb
```

**使用场景：** 定期检查机顶盒状态

### 模式4：清空POS数据
清空所有POS设备记录和处理记录，不导入新数据。

**Windows:**
```cmd
python import_equipment_data.py --clear-pos
```

**Linux/Mac:**
```bash
python3 import_equipment_data.py --clear-pos
```

**使用场景：** 今天不需要展示POS数据

### 模式5：清空机顶盒数据
清空所有机顶盒设备记录和处理记录，不导入新数据。

**Windows:**
```cmd
python import_equipment_data.py --clear-stb
```

**Linux/Mac:**
```bash
python3 import_equipment_data.py --clear-stb
```

**使用场景：** 今天不需要展示机顶盒数据

## 典型使用场景

### 场景1：每日POS检查
每天只需要检查收银系统，不需要更新机顶盒数据：

1. 上传最新的收银设备文件和在营门店文件
2. 运行：`python3 import_equipment_data.py --only-pos`
3. 页面会显示最新的POS状态，机顶盒数据保持不变

### 场景2：周度机顶盒检查
每周检查一次机顶盒，不影响每日的POS数据：

1. 上传最新的机顶盒文件和在营门店文件
2. 运行：`python3 import_equipment_data.py --only-stb`
3. 页面会显示最新的机顶盒状态，POS数据保持不变

### 场景3：临时隐藏某类设备
今天不想展示POS数据，只看机顶盒：

1. 运行：`python3 import_equipment_data.py --clear-pos`
2. 页面只显示机顶盒数据

### 场景4：全量更新
需要同时更新所有设备数据：

1. 上传所有文件（收银设备、机顶盒、在营门店）
2. 运行：`python3 import_equipment_data.py`
3. 所有数据都会更新

## 数据处理逻辑

### 导入规则
1. **营业状态过滤**：只导入营业状态为"营业中"的门店
2. **白名单匹配**：只导入在白名单中的门店（用于匹配战区和区域经理）
3. **设备状态过滤**：只导入状态为"离线"的设备

### 数据清理规则
- `--only-pos`：清空旧POS数据和处理记录，保留机顶盒数据
- `--only-stb`：清空旧机顶盒数据和处理记录，保留POS数据
- 默认模式：清空所有旧数据和处理记录
- `--clear-pos`：只清空POS数据，不导入
- `--clear-stb`：只清空机顶盒数据，不导入

### 处理记录说明
每次导入会清空对应设备类型的处理记录（已恢复/未恢复原因），这样可以：
- 避免旧的处理记录影响新数据
- 确保每次导入都是全新的状态
- 区域经理需要重新处理新导入的设备

## 注意事项

1. **文件位置**：确保文件在 `equipment_status/` 目录下
2. **文件格式**：确保Excel文件格式正确，列位置匹配
3. **数据库连接**：确保数据库连接正常
4. **权限问题**：Linux环境需要执行权限（`chmod +x *.sh`）
5. **处理记录**：导入新数据会清空对应类型的处理记录

## 常见问题

### Q: 导入后没有数据？
A: 检查：
1. 文件是否在 `equipment_status/` 目录
2. 文件名格式是否正确
3. 门店是否在白名单中
4. 门店营业状态是否为"营业中"
5. 设备状态是否为"离线"

### Q: 如何只更新POS不影响机顶盒？
A: 使用 `--only-pos` 参数：
```bash
python3 import_equipment_data.py --only-pos
```

### Q: 今天不想显示某类设备怎么办？
A: 使用清空命令：
```bash
# 不显示POS
python3 import_equipment_data.py --clear-pos

# 不显示机顶盒
python3 import_equipment_data.py --clear-stb
```

### Q: 如何更新白名单？
A: 参考《更新白名单完整指南.md》

### Q: 处理记录会保留吗？
A: 不会。每次导入对应类型的设备数据时，会清空该类型的处理记录，确保数据一致性。
