# 服务器部署注意事项

## 数据库配置差异

### 本地测试（当前配置）
- 使用SQLite数据库
- 文件：`local_test.db`
- 配置位置：
  - `viewer/app_viewer.py` 第48行
  - `import_equipment_data.py` 第21行

### 服务器部署
需要修改为PostgreSQL配置：

#### 1. viewer/app_viewer.py
```python
# 数据库配置
DATABASE_URL = os.getenv(
    'DATABASE_URL',
    'postgresql://postgres:postgres@127.0.0.1:5432/configurable_ops'
)
```

#### 2. import_equipment_data.py
```python
# 数据库配置
DATABASE_URL = os.getenv(
    'DATABASE_URL',
    'postgresql://postgres:postgres@127.0.0.1:5432/configurable_ops'
)
```

## 快速切换方法

### 方法1：使用环境变量（推荐）
在服务器上设置环境变量，无需修改代码：

```bash
export DATABASE_URL="postgresql://postgres:postgres@127.0.0.1:5432/configurable_ops"
```

### 方法2：修改代码
上传到服务器前，修改两个文件中的DATABASE_URL默认值。

## 部署步骤

### 1. 准备文件

- `viewer/app_viewer.py`
- `import_equipment_data.py`

### 2. 上传文件到服务器
```bash
# 上传新文件

scp viewer/static/equipment.* root@your-server:/var/www/nyb_weekly_check_tool_1.0/viewer/static/
```

### 3. 上传设备数据
```bash
# 创建文件夹
ssh root@your-server "mkdir -p /var/www/nyb_weekly_check_tool_1.0/equipment_status"
```

### 4. 在服务器上执行
```bash

ssh root@your-server

# 导入设备数据
python3 import_equipment_data.py

# 重启服务
systemctl restart review-viewer

# 查看状态
systemctl status review-viewer
```

## 常见问题

### Q: 表不存在错误
A: 运行 `python3 import_equipment_data.py`，脚本会自动创建表

### Q: 数据为空
A: 检查whitelist是否已导入，设备数据需要匹配whitelist

### Q: 页面404
A: 检查服务是否重启，查看服务日志

### Q: 筛选选项为空
A: 确认数据已成功导入，检查数据库连接
