# 营业状态逻辑变更说明

## 变更概述

将设备异常筛选逻辑从"排除暂停营业门店"改为"只保留营业中门店"。

## 旧逻辑 vs 新逻辑

### 旧逻辑（黑名单模式）
```
1. 读取"门店档案"文件
2. 获取所有暂停营业的门店ID
3. 导入设备异常时，排除暂停营业的门店
4. 展示：所有设备异常 - 暂停营业门店
```

**问题**：
- 依赖"门店档案"文件的准确性
- 黑名单模式，容易遗漏

### 新逻辑（白名单模式）
```
1. 读取"在营门店"文件（文件名包含"在营门店"）
2. 从"营业门店" sheet 读取营业中的门店ID
3. 导入设备异常时，只保留在营门店列表中的门店
4. 展示：设备异常 ∩ 营业中门店
```

**优势**：
- 白名单模式，更准确
- 数据源更可靠
- 逻辑更清晰

## 文件格式要求

### 在营门店文件
- **文件位置**：`equipment_status/` 文件夹
- **文件名**：包含"在营门店"关键字（如：`在营门店_20260305.xlsx`）
- **Sheet名称**：`营业门店`（第一个sheet）
- **列定义**：
  - C列（第3列）：门店ID
  - I列（第9列）：营业状态（应该都是"营业中"）

### 示例数据
```
| A | B | C (门店ID) | D | E | F | G | H | I (营业状态) |
|---|---|-----------|---|---|---|---|---|-------------|
| ... | ... | 1001 | ... | ... | ... | ... | ... | 营业中 |
| ... | ... | 1002 | ... | ... | ... | ... | ... | 营业中 |
| ... | ... | 1003 | ... | ... | ... | ... | ... | 营业中 |
```

## 代码修改

### 修改文件
`import_equipment_data.py`

### 主要变更

#### 1. 文件查找逻辑
```python
# 旧代码
store_files = list(equipment_dir.glob('牛约堡集团_门店档案*.xlsx'))
store_file = max(store_files, key=lambda p: p.stat().st_mtime)

# 新代码
operating_store_files = list(equipment_dir.glob('*在营门店*.xlsx'))
operating_store_file = max(operating_store_files, key=lambda p: p.stat().st_mtime)
```

#### 2. 数据读取逻辑
```python
# 旧代码 - 读取暂停营业门店（黑名单）
df_closed = pd.read_excel(store_file, header=2)
closed_stores = set(str(sid) for sid in df_closed['机构编码'].dropna())

# 新代码 - 读取营业中门店（白名单）
df_operating = pd.read_excel(operating_store_file, sheet_name='营业门店')
operating_stores = set()
for _, row in df_operating.iterrows():
    store_id = str(row.iloc[2]).strip()  # C列（索引2）
    status = str(row.iloc[8]).strip()     # I列（索引8）
    if status == '营业中':
        operating_stores.add(store_id)
```

#### 3. 筛选逻辑
```python
# 旧代码 - 排除暂停营业门店
if store_id in closed_stores:
    skip_closed += 1
    continue

# 新代码 - 只保留营业中门店
if store_id not in operating_stores:
    skip_not_operating += 1
    continue
```

## 注意事项

### 1. 文件命名
- 在营门店文件名必须包含"在营门店"关键字
- 如果有多个文件，会自动选择最新的

### 2. Sheet名称
- 必须是"营业门店"（第一个sheet）
- 如果sheet名称不对，会报错

### 3. 列位置
- C列（索引2）：门店ID
- I列（索引8）：营业状态
- 列位置固定，不要调整

### 4. 营业状态值
- 必须是"营业中"（精确匹配）
- 其他值（如"暂停营业"、"筹备中"等）会被忽略

### 5. 数据清理
- 门店ID会自动去除首尾空格
- 营业状态会自动去除首尾空格

## 常见问题

### Q1: 找不到在营门店文件
**A**: 检查文件名是否包含"在营门店"关键字，文件是否在 `equipment_status/` 文件夹中。

### Q2: Sheet名称错误
**A**: 确保第一个sheet名称是"营业门店"，不是其他名称。

### Q3: 导入的门店数量为0
**A**: 检查：
- I列营业状态是否都是"营业中"
- C列门店ID是否正确
- 门店ID是否在whitelist中

### Q4: 导入的门店数量比预期少
**A**: 可能原因：
- 部分门店不在whitelist中（会跳过）
- 部分门店营业状态不是"营业中"
- 部分门店没有设备异常
