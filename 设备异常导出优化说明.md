# 设备异常导出功能优化说明

## 优化内容

### 新增导出字段

原有字段：
- 门店ID
- 门店名称
- 战区
- 区域经理
- 设备类型
- 设备编号
- 设备名称
- 当前状态
- 处理动作
- 未恢复原因
- 处理时间

新增字段（用于统计和处罚）：
1. **是否经常出问题**：是/否
2. **触发原因**：当天反复/5天3次/10天4次
3. **最近5天异常次数**：数字（仅POS）
4. **最近10天异常次数**：数字（仅POS）
5. **异常时间点**：最近5次的具体时间（如：03-09下午、03-09上午、03-08下午）
6. **预计恢复日期**：区域经理填写的预计恢复时间

## 使用场景

### 1. 统计经常出问题的门店
筛选条件：`是否经常出问题 = 是`

用途：
- 统计问题门店数量
- 按战区/区域经理分组统计
- 识别需要重点关注的门店

### 2. 处罚依据
根据以下字段判断：
- **触发原因**：
  - "当天反复" = 最严重（上午处理了下午又出问题）
  - "5天3次" = 较严重（短期内频繁出问题）
  - "10天4次" = 一般严重（中期内多次出问题）

- **异常时间点**：
  - 查看具体哪些时间点出问题
  - 判断是否有规律（如每天下午）
  - 作为处罚的证据

### 3. 区域经理绩效考核
按区域经理分组统计：
- 负责门店总数
- 经常出问题的门店数
- 问题门店占比
- 平均处理时间
- 预计恢复日期准确率

### 4. 设备更换决策
根据以下信息判断是否需要更换设备：
- 最近10天异常次数 ≥ 5次
- 触发原因 = "当天反复"
- 未恢复原因包含"设备老化"、"需要更换"等关键词

## 数据说明

### 异常时间点格式
- 格式：`MM-DD上午` 或 `MM-DD下午`
- 示例：`03-09下午、03-09上午、03-08下午`
- 只显示最近5次（从新到旧）
- 用顿号（、）分隔

### 统计范围
- 最近5天：从今天往前推5个自然日
- 最近10天：从今天往前推10个自然日
- 只统计有快照记录的日期（缺失日期自动忽略）

### 机顶盒说明
- 机顶盒不启用历史追踪
- 相关统计字段为空
- 只显示当前状态和处理信息

## 注意事项

1. **首次导出可能较慢**
   - 需要查询所有门店的快照历史
   - 需要计算经常出问题统计
   - 建议在非高峰期导出

2. **数据准确性**
   - 统计基于快照数据
   - 如果快照缺失，统计会不准确
   - 建议每天按时导入数据

3. **Excel文件大小**
   - 新增字段会增加文件大小
   - 但影响不大（每行增加约100字节）

4. **兼容性**
   - 使用openpyxl引擎
   - 兼容Excel 2007及以上版本
   - 支持中文文件名
